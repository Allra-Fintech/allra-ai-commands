# GitHub Pull Request Generator

현재 브랜치의 변경사항을 분석하여 develop 브랜치로 보낼 GitHub Pull Request를 자동 생성 후 원격 저장소에 업로드

## 현재 작업 상태 분석

```bash
git branch --show-current                    # 현재 브랜치명
git fetch origin --prune                     # 원격 최신 상태 갱신
git diff origin/develop --name-only          # origin/develop과 비교한 수정 파일명
git diff origin/develop --stat               # origin/develop과 비교한 파일별 변경 라인 수
git log origin/develop..HEAD --oneline       # origin/develop 이후 추가된 커밋들
git diff origin/develop                      # origin/develop과의 상세 차이점 (필요시)
```

## 📋 PR 작성 가이드라인

1. 불필요한 섹션은 생략
2. "기능 추가"가 아닌 "사용자 프로필 수정 기능 추가"
3. 실제 확인 가능한 체크리스트만 포함
4. 브랜치명과 커밋 메시지에서 맥락 파악

### 1. 제목 작성 규칙

다음 중 하나의 접두사를 사용하여 제목을 작성하세요:

- `[🚀 Release]` - 릴리즈 관련
- `[✨ Feature]` - 새로운 기능 추가
- `[🐛 Fix]` - 버그 수정
- `[🚑 HotFix]` - 긴급 핫픽스
- `[♻️ Refactor]` - 코드 리팩토링
- `[💄 DESIGN Q/A]` - 디자인 QA
- `[🧪 TestCase]` - 테스트 케이스
- `[📝 Docs]` - 문서 수정
- `[🔧 Config]` - 설정 파일 수정
- `[📦 Dependencies]` - 의존성 업데이트

### 2. PR 템플릿 구조

```markdown
## ✨ 주요 변경 사항

- <변경된 사항 요약 (예: 새로운 기능 추가, 버그 수정 등)>

## ✅ 체크리스트

- [ ] 모든 기능이 정상적으로 동작하고 있나요?
- [ ] 테스트 케이스를 모두 통과했나요?
- [ ] 배포 관련 설정이 문제없이 적용되었나요?
- [ ] 필요한 모든 리뷰어가 지정되었나요?

## 📄 관련 이슈 및 참고 사항

- 이슈 번호: #(이슈 번호 작성)
- 추가 참고 사항: <필요 시 추가 설명>
- close #(이슈 번호 작성)

## 🔍 변경 이유 및 배경

- <변경된 사항의 목적과 이유>
- <배포의 필요성 및 주요 기능 설명>

## 🧪 테스트 및 검증

- <테스트 방법과 결과>
- <배포 전 확인해야 할 사항 (예: 환경 변수 설정, 외부 API 변경 등)>

## 📝 배포 후 확인 사항

- [ ] 배포 후 확인해야 할 사항
  - <특정 기능의 정상 작동 여부 확인>
  - <모니터링 설정>

## 🛠 추가 변경 사항

- <추가로 수정된 코드나 개선된 부분이 있다면 설명>
```

### 3. 분석 방법

1. **현재 브랜치 확인**: `git branch` 또는 `git status`로 현재 브랜치 파악
2. **변경사항 분석**: `git diff` 또는 `git log`로 변경된 파일들 확인
3. **파일 유형별 분류**:
   - `.tsx`, `.ts`, `.jsx`, `.js` → 기능/컴포넌트 관련
   - `.css`, `.scss` → 스타일 관련
   - `.md`, `.txt` → 문서 관련
   - `.json`, `.config.js` → 설정 관련
   - 테스트 파일 → 테스트 관련

### 4. 제목 선택 기준

- **새로운 기능 추가** → `[✨ Feature]`
- **버그 수정** → `[🐛 Fix]`
- **UI/UX 개선** → `[💄 DESIGN Q/A]`
- **코드 정리/최적화** → `[♻️ Refactor]`
- **긴급 수정** → `[🚑 HotFix]`
- **릴리즈 준비** → `[🚀 Release]`
- **테스트 추가** → `[🧪 TestCase]`

### 5. 주요 변경 사항 작성 팁

- 변경된 파일들의 핵심 기능을 요약
- 사용자에게 미치는 영향 설명
- 기술적 변경사항 간단히 언급
- 새로운 의존성이나 설정 변경사항 포함

### 6. 체크리스트 커스터마이징

변경사항에 따라 체크리스트를 조정하세요:

- **기능 추가 시**: 기능 동작 확인, 테스트 케이스 통과
- **버그 수정 시**: 버그 재현 불가, 관련 기능 정상 동작
- **리팩토링 시**: 기존 기능 동작 확인, 성능 개선 검증
- **UI 변경 시**: 디자인 가이드 준수, 반응형 확인

### 7. 예시

```markdown
# [✨ Feature] 사용자 프로필 수정 기능 추가

## ✨ 주요 변경 사항

- 사용자 프로필 수정 페이지 구현
- 프로필 이미지 업로드 기능 추가
- 개인정보 수정 API 연동

## ✅ 체크리스트

- [x] 프로필 수정 기능이 정상적으로 동작하고 있나요?
- [x] 이미지 업로드가 올바르게 처리되나요?
- [x] API 연동이 정상적으로 작동하나요?
- [ ] 필요한 모든 리뷰어가 지정되었나요?

## 📄 관련 이슈 및 참고 사항

- 이슈 번호: #123
- 추가 참고 사항: 디자인 시스템 가이드 준수

## 🔍 변경 이유 및 배경

- 사용자 요청에 따른 프로필 수정 기능 필요
- 기존 프로필 조회 기능과의 일관성 유지

## 🧪 테스트 및 검증

- 프로필 수정 페이지 접근 및 동작 확인
- 이미지 업로드 및 미리보기 기능 테스트
- API 응답 및 에러 처리 검증

## 📝 배포 후 확인 사항

- [ ] 프로필 수정 기능 정상 동작 확인
- [ ] 이미지 업로드 서버 저장 확인
- [ ] 모바일 환경에서의 UI 확인

## 🛠 추가 변경 사항

- 프로필 수정 시 유효성 검사 로직 추가
- 이미지 압축 및 최적화 기능 구현
```
