# GIT Commit Message Generator

1. 현재 git 변경사항을 분석하여 적절한 깃모지와 함께 커밋 메시지를 작성해주세요.
2. 다음 깃모지 가이드라인을 따라주세요:

## 깃모지 가이드라인:

- ✨ : 새로운 기능 추가
- 🐛 : 버그 수정
- 🎨 : 코드 구조/형식 개선 (가독성/정리)
- ♻️ : 코드 리팩토링
- ⚡️ : 성능 개선
- 🔥 : 코드/파일 삭제
- 📝 : 문서 작성/수정
- 🚀 : 배포 관련
- ✅ : 테스트 추가/수정
- 🔧 : 설정 파일 수정
- 📦 : 패키징/배포 산출물 변경
- 🔒 : 보안 이슈 수정
- 🌐 : 국제화/다국어 지원
- 💄 : UI/스타일 변경 (디자인/레이아웃/뷰)
- 🔀 : 브랜치 병합
- ⬆️ : 의존성 업그레이드
- 🏷️ : 타입 추가/수정
- 🚧 : 작업 진행 중
- 💚 : CI 빌드 수정
- 👷 : CI/CD 시스템 추가/수정
- 📱 : 반응형 디자인
- 🗃️ : 데이터베이스 관련
- 🔊 : 로그 추가
- 🔇 : 로그 제거
- 🩹 : 간단한 수정
- 🏗️ : 아키텍처 변경
- 💡 : 주석 추가/수정
- 🍱 : 에셋 추가/수정
- ♿️ : 접근성 개선
- 🎉 : 프로젝트 시작
- 🔖 : 릴리즈/버전 태그
- 🚨 : 컴파일러/린터 경고 수정
- 🙈 : .gitignore 추가/수정

## 커밋 메시지 형식:

`:깃모지: 간단명료한 한국어 설명`

예시:

- `:art: 홈페이지 UI 변경 및 알림 API 작성`
- `:sparkles: 사용자 로그인 기능 구현`
- `:bug: 로그인 시 토큰 만료 오류 수정`

## 분석 방법:

1. `git status`와 `git diff`를 확인하여 변경된 파일들을 파악
2. 변경된 파일의 확장자와 내용을 바탕으로 적절한 깃모지 선택
3. 변경사항의 핵심을 간단명료하게 요약
4. 한국어로 자연스럽게 작성

## 파일 확장자별 우선순위:

- `.css`, `.scss`, `.vue`, `.jsx`, `.tsx` → 💄 (UI 관련)
- `.test.js`, `.test.ts`, `.test.tsx`, `.spec.ts`, `.spec.tsx`, `.cy.ts`, `.e2e.ts` → ✅ (테스트)
- `.md`, `.txt` → 📝 (문서)
- `.json`, `.yaml`, `.config.js` → 🔧 (설정)
- API 관련 파일 → ✨ (신규 기능) 또는 ♻️ (리팩토링)
- 컴포넌트 파일 → 💄 (UI 변경) 또는 ✨ (기능 추가)

변경사항이 여러 종류일 경우 가장 중요한 변경사항을 기준으로 선택하고,
추가 변경사항을 간단히 언급해주세요.
