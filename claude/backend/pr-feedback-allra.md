# PR 피드백 자동 처리 (Allra)

현재 브랜치의 PR에 대해 CI 실패 수정 → CodeRabbit 리뷰 코멘트 처리 → PR 승인 대기까지 자동으로 반복 수행합니다.
모든 GitHub 작업은 `gh` CLI를 사용합니다.

## 인자
- `$ARGUMENTS`: PR 번호 (생략 시 현재 브랜치의 PR 자동 탐지)

## 워크플로우

### 0단계: PR 탐지

```bash
# 현재 브랜치의 PR 번호 탐지
gh pr list --head $(git branch --show-current) --json number --jq '.[0].number'

# owner/repo 확인
gh repo view --json owner,name --jq '"\(.owner.login)/\(.name)"'

# PR 기본 정보 확인
gh pr view {prNumber} --json title,state,reviewDecision,statusCheckRollup
```

### 1단계: CI 워크플로우 확인 및 수정

```bash
# CI check 상태 확인
gh pr checks {prNumber}

# 실패한 workflow run의 로그 확인
gh run view {runId} --log-failed
```

- **실패한 check가 있는 경우:**
  - `gh run view {runId} --log-failed`로 실패 원인 분석
  - 코드 수정 → 로컬에서 해당 check 재현 확인:
    - sonarlintMain: `./gradlew sonarlintMain`
    - sonarlintTest: `./gradlew sonarlintTest`
    - spotlessCheck: `./gradlew spotlessApply`
    - test: `SPRING_PROFILES_ACTIVE=test ./gradlew test`
    - build: `./gradlew build`
  - 수정사항 커밋 & `git push`
- **전체 pass인 경우:** 2단계로 진행
- **pending/running 상태:** 사용자에게 알리고 대기 여부 확인

### 2단계: CodeRabbit 리뷰 코멘트 수집

```bash
# unresolved 리뷰 스레드 조회 (GraphQL)
gh api graphql -f query='
{
  repository(owner: "{owner}", name: "{repo}") {
    pullRequest(number: {prNumber}) {
      reviewThreads(first: 50) {
        nodes {
          id
          isResolved
          comments(first: 5) {
            nodes {
              databaseId
              body
              path
              line
              author { login }
            }
          }
        }
      }
    }
  }
}'
```

- isResolved=true인 스레드는 제외
- unresolved 코멘트가 없으면 5단계로 진행

### 3단계: 코멘트 분석 및 자동 처리

각 unresolved 코멘트에 대해 **스스로 판단**하여 처리:

#### 반영이 필요한 경우 (코드 품질, 버그, 컨벤션 위반 등)
1. 해당 파일 읽기 → 코드 수정
2. 수정사항 커밋
3. 수정 내용을 요약하여 해당 코멘트에 **대댓글** 등록:
```bash
gh api repos/{owner}/{repo}/pulls/{prNumber}/comments \
  -X POST \
  -F in_reply_to={commentId} \
  -f body='수정 반영했습니다. {수정 내용 요약}'
```
4. 해당 리뷰 스레드 resolve:
```bash
gh api graphql -f query='
mutation {
  resolveReviewThread(input: {threadId: "{threadId}"}) {
    thread { isResolved }
  }
}'
```

#### 반영하지 않는 경우 (오탐, 선택사항, 비즈니스 의도 등)
1. 해당 코멘트에 **짧은 미변경 사유** 대댓글 등록:
```bash
gh api repos/{owner}/{repo}/pulls/{prNumber}/comments \
  -X POST \
  -F in_reply_to={commentId} \
  -f body='{미변경 사유}'
```
2. 해당 리뷰 스레드 resolve:
```bash
gh api graphql -f query='
mutation {
  resolveReviewThread(input: {threadId: "{threadId}"}) {
    thread { isResolved }
  }
}'
```

#### 판단이 어려운 경우 (비즈니스 로직, 설계 방향 등)
- 사용자에게 해당 코멘트 내용과 선택지를 보여주고 확인 받기

### 4단계: 수정사항 push

```bash
# 포맷팅 적용
./gradlew spotlessApply

# 로컬 CI 검증
SPRING_PROFILES_ACTIVE=test ./gradlew test spotlessCheck sonarlintMain sonarlintTest

# push
git push
```

- 검증 실패 시 수정 후 재시도

### 5단계: PR 승인 상태 확인

```bash
# 승인 상태 확인
gh pr view {prNumber} --json reviewDecision --jq '.reviewDecision'

# PR 전체 상태 요약
gh pr view {prNumber} --json state,reviewDecision,statusCheckRollup --jq '{state, reviewDecision, checks: [.statusCheckRollup[] | {name: .name, status: .status, conclusion: .conclusion}]}'
```

- **APPROVED**: 사용자에게 승인 완료 알림 후 종료. 반복 루프도 종료.
- **CHANGES_REQUESTED / REVIEW_REQUIRED**:
  - 처리할 코멘트나 CI 실패가 남아있으면 즉시 1단계부터 반복
  - 모든 코멘트 resolved + CI pass인데 아직 미승인이면, `/loop` 동적 모드로 1분 간격 재확인 반복
  - **반드시 APPROVED 될 때까지 자동 반복한다.** 사용자에게 중단 여부를 묻지 않고 계속 진행.

## 커밋 규칙
- 커밋 메시지: 프로젝트 컨벤션 준수 (`[카테고리] 영문 제목`)
- CI 수정: `[🐛 Fix] Resolve CI failure - {원인 요약}`
- 리뷰 반영: `[🐛 Fix] Address PR review feedback`
- **저자/저작권 정보 절대 포함 금지**: `Co-Authored-By`, `Author`, `Authored-by`, `Generated by`, 저작권자, 서명 등 저자 및 저작권 관련 문구를 커밋 메시지에 일체 넣지 않음
- spotlessApply 후 커밋

## 주의사항
- 비즈니스 로직 변경이 필요한 코멘트는 반드시 사용자 확인
- push 전 반드시 로컬 CI 검증 통과 확인
- 이미 resolved된 코멘트는 건너뛰기
- coderabbitai 외 다른 리뷰어의 코멘트도 동일하게 처리

## 사용 예시
```
/pr-feedback-allra           # 현재 브랜치 PR 자동 처리
/pr-feedback-allra 165       # PR #165 자동 처리
```
