# Linear 이슈 조회

Linear MCP로 현재 이슈를 빠르게 조회하는 기본 슬래시 커맨드입니다. 
주요 목적은 **조회 위주 워크플로우**로, 조회 결과를 바로 판단/선택할 수 있도록 하는 것입니다.

## 인자
- `$ARGUMENTS`: 슬래시 커맨드 뒤의 추가 인자.
  - 단독 사용 시 기본 동작을 수행: 내 담당 오픈 이슈(최근 업데이트 순) 조회
- `-t, --team <팀명|팀ID>`: 팀 식별자
- `-p, --project <프로젝트명|프로젝트ID>`: 프로젝트 식별자
- `-a, --assignee <사용자|me>`: 담당자. `me`는 로그인 사용자
- `-s, --state <상태>`: 상태 (예: `open`, `in progress`, `todo`, `done`)
- `-l, --label <라벨>`: 라벨 필터
- `-q, --query <검색어>`: 제목/내용 검색어
- `-n, --limit <숫자>`: 조회 개수 (기본 `20`, 최대 `100`)
- `--sort <컬럼>`: 정렬 기준, 기본 `updatedAt desc`
- `-f, --format <table|json|compact|md>`: 출력 형식

## 사용법
- `/linear`
  - 팀 미지정이면 현재 사용자가 속한 팀을 후보로 보여주고 하나 선택
  - assignee 미지정 → `me`
  - state 미지정 → `open` 계열 기본값으로 필터
  - limit 기본값 `20`
  - 결과는 `[ID] [Title] [State] [Assignee] [Updated] [URL]` 테이블로 출력

- `/linear -t platform -a me -n 30`
  - platform 팀, 내 담당, 최근 30건 조회

- `/linear -q "로그인" -s in progress -n 10`
  - "로그인" 검색어가 들어간 진행중 이슈 10건 조회

- `/linear -a me -f json`
  - 자동 처리/스크립팅 용도로 JSON 형식 출력

- `/linear -s blocked`
  - 상태만 blocked로 지정해서 조회

## 워크플로우

1. 입력 파싱
   - 위 인자를 파싱하고, 누락/모호한 값은 보완 여부 판단

2. 모호성 보정
   - `team` 미지정: 팀 후보 조회
     - 후보가 1개면 자동 선택
     - 2개 이상이면 사용자에게 선택 지시
   - `assignee=me`면 현재 사용자 식별값으로 변환
   - `state`가 비어 있으면 `open` 계열 기본값 적용

3. 조회 실행
   - `linear.list_issues`(또는 동등한 MCP 액션)로 조회
   - 필터 조건: team/project/assignee/state/label/query
   - 정렬은 `updatedAt desc`

4. 결과 정리
   - 기본: 사람이 보기 쉬운 compact 테이블
   - 상태/담당자/업데이트일로 빠른 스캔이 가능하게 표시
   - 포맷 옵션:
     - `table` (기본): 보기용 요약
     - `compact`: 칼럼 축소형 요약
     - `md`: 마크다운 표
     - `json`: 원본 데이터 정렬형 출력

5. 조회 결과 해석 가이드
   - 빈 결과: "조건을 넓혀 조회" 제안
   - 에러: 인증 실패, 팀 미해결, 필터 미일치 시 해결 방안 제시

## 커맨드 규칙(중요)
- 기본은 오직 조회이며, 이슈 생성/수정/상태변경은 하지 않음
- 응답은 간결하게, 먼저 상위 중요 정보(현재 담당자/상태/최근업데이트) 노출
- 기본 조회는 1회만 추가 질문, 나머지는 추정값 사용 금지(명시적 값 필요 시 질의)
- 동일 요청은 동일 정렬/형식으로 일관되게 제공

## 별칭 제안
- `/linear` (기본 조회)
- `/linear-mine` (shorthand: 내 담당 오픈 이슈)
- `/linear-search` (query 중심 조회)

이 문서에서는 단일 커맨드 `/linear` 구현을 중심으로 작성하고, 위 별칭은 필요 시 추후 분기 커맨드로 분리 가능.
